Java 面向对象

入门

在Java中 对象 其实就是一个 Class（类），在Java世界中所有的程序都是由Class（对象、类）组成的。为了更好的理解对象这件事，有一个专门的图形语言来管理和维护对象，这个语言就是 UML 。

类的关联关系

对象是对问题领域中事物的抽象。

所谓抽象便是将事物变成编程语言的过程：

  1. 大学生信息可以抽象为 Student 对象，在Java中可以创建一个 class Student
  2. 大学专业可以抽象为 Major对象，在Java中我们可以创建一个 class Major

对象特性：

  1. 万物皆对象。

    所有的事物都可以抽象成对象。

  2. 每个对象都是唯一的。

    世界上不存在一模一样的人，所以在编程时抽象出来的对象也是唯一的。

    注：对象的唯一是指类实例化后的每个实例都是唯一的，对于相同事物抽象后的类可以是相同的。

  3. 对象具备属性和方法。

    每一个对象都拥有自己的属性定义和值，属性+方法 对象便是完整的。

  • 如果是绿色圆形图标（不管图标是否实心），那么说明当前的访问控制修饰符是 public

注：颜色不重要，图形才是重要的。不同的 UML 软件颜色会有差异。

封装

定义：隐藏对象属性，只能通过公开的方法得到或者设置属性值。业界统一规范：getter、setter

  • get + 属性名称（属性名的首字母要大写），用于获取属性
  • set + 属性名称（属性名的首字母要大写），用于设置属性

常量

实例变量可以很好的支持面向对象的封装，但是有些时间我们还是需要在多个对象里共享一些数据，这个时候就需要引入 常量 了，常量 也可以成为 静态变量。

常量应用实例：（存储和调用学校记录）

当我们定义了 静态变量 后，就可以直接操作这个 schools 变量了

常量的特点之一就是不需要实例化，在Java 中实际上是创建了一个全局唯一的内存空间并且分配给了这个常量。所以常量值只会随着 Java 的销毁而销毁，这样也就方便我们存储数据到内存中，否则数据会随着对象一起被回收。

常量使用常见场景：

  1. 常用的字符串值
  2. 需要在内存做缓存的值

为了和变量区分开，一般情况下常量的变量名采用全大写字母来定义，单词之间使用 “_” 符号来分割。
