Housekeeping

“Unsafe” programming is one of the major culprits that makes programming
expensive.

This Keyword

The this keyword is often used in return statements for returning the reference to the current object(class Leaf) also useful for passing the current object to another method(class Apple).

Some people will obsessively put this in front of every method call and field reference, arguing that it makes it “clearer and more explicit.” Don’t do it. There’s a reason that we use high-level languages. They do things for us. If you use this when it’s not necessary, you confuse and annoy everyone who reads your code because all the rest of the code they’ve read won’t use this everywhere. People expect this only when it is necessary. Following a consistent and straightforward coding style saves time and money.

Finalization and Garbage Collection

Java does have the garbage collector to reclaim the memory of objects that no longer used.

When the garbage collector is ready to release the storage used for your object, if first calls finalize(), and only on the next garbage-collection pass will it reclaim the object memory.

In C++, objects always get destroyed, whereas in Java, objects do not always get garbage collected.

Three points to remember:

  • Your object might not get garbage collected.
  • Garbage collection is not destruction.
  • Garbage collection is only about memory.

Must perform cleanup. If the C++ object is created as a local (i.e., on the stack—not
possible in Java), the destruction happens at the closing curly brace of the scope
where the object was created. If the C++ programmer forgets to call delete, the destructor
is never called, and you have a memory leak, plus the other parts of the object never
get cleaned up.

In contrast, Java must always use new to create object. The garbage collector will releases the storage automatically.

Neither garbage collection nor finalization is guaranteed. If the JVM isn’t close to running out of memory, it might not waste time recovering memory through garbage collection.

The Termination Condition. If any portions of the object are not properly cleaned up, you have a bug in your program that can be very difficult to find. finalize() can eventually discover this condition, even if it isn’t always called.

In java, the garbage collector can significantly increase the speed of object creation.

Garbage-collection technique:

  • Reference Counting

    Each object contains a reference counter, and every time a reference is attached to that object, the reference count is increased. Every time a reference goes out of scope or is set to null, the reference count is decreased.

    The garbage collector moves through the entire list of objects, and when it finds one with a reference count of zero it releases that storage

    Drawback: If objects circularly refer to each other they can have nonzero reference counts while still being garbage.
